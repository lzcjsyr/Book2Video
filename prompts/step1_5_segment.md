# 步骤 1.5 Agent 分段与画面模式选择

你是视频脚本分段 Agent。请读取下面的 raw JSON 文件，并把口播正文分成适合视频呈现的段落，同时为每段选择画面呈现模式。

输入文件：
{raw_json_path}

输出文件：
{output_json_path}

目标段数：
{target_segments}

## 输出格式

只写入一个 JSON 文件，结构必须如下：

```json
{
  "segments": [
    {
      "content": "这一段完整口播原文",
      "visualizer": "image"
    }
  ]
}
```

`visualizer` 只能是：
- `image`：适合静态图像生成。
- `hf`：适合 HyperFrames 动态配图。

## 分段原则

1. 不要在句子内部切分。不要把同一个句号、问号、感叹号或分号结束前的内容拆成几段。
2. 一个意群完整了就可以切一段。每段要有完整画面，观众能直接理解这一段要呈现什么。
3. 数据多、逻辑性强、抽象关系强、因果链条强的段落，优先用 `hf`。
4. 画面感强、故事性强、人物动作或场景情绪明显的段落，优先用 `image`。
5. 不要改写、增删、总结或润色原文。所有段落按顺序拼接后，必须与 raw JSON 的 `content` 完全一致（忽略空白差异）。
6. 目标段数是参考值，不要为了凑数破坏句子或意群。

完成后只需要写入输出 JSON 文件。
