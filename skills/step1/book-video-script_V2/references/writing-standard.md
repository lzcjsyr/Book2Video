# Enhanced Video Book Script Writing Standard

把源资料转成适合抖音和微信视频号的中文音频解说稿。目标不是复述资料，也不是读书报告，而是替中国观众做几分钟内的“第一遍速读”：讲清最值得知道的是什么、为什么重要、和自己有什么关系，以及为什么值得继续读或继续聊。

完整执行顺序见入口 `SKILL.md` 的“主流程”。本文定义角度判断、读书视频原型、写作标准和最终 JSON 契约。

## 最高优先级

- 忠于资料：不虚构、不偷换、不强加原资料未表达的立场。
- 先讲清“是什么、为什么重要”，再追求金句和传播感。
- 找到一个叙事主线。不机械总结，不按章节或目录平推。
- 口吻要像一个读透资料、也懂普通人处境的人在讲。
- 让观众觉得“这事和我有关”：有冲突、反常识、社交货币和讨论空间。
- 有人文关怀。不要为了传播感制造廉价对立。

## 角度判断

写正文前先完成 `_angle_brief.json`。不要直接从资料摘要跳到初稿。

### HKR 质检

用 Cody 的 HKR 方法判断角度，但改造成读书视频语境：

- **H - Hook**：是否有悬念、冲突、反常识或现实困惑，让观众愿意停下来听。
- **K - Knowledge**：是否有资料真正提供的新知识、新框架、新因果解释或新判断。
- **R - Resonance**：是否能照到普通人的工作、关系、选择、焦虑、机会、风险或价值冲突。

优先选择三项都成立的角度。至少要满足 K 和 H/R 之一。如果只有知识量，没有悬念和共鸣，要重新找现实切口；如果只有情绪，没有资料支撑，要收回夸张表达。

### 角度 brief 字段

保存到最终 raw JSON 同一目录：

```json
{
  "source_name": "《作品名》",
  "core_question": "这份资料真正回答的一个问题",
  "viewer_relevance": "这个问题为什么和普通观众有关",
  "hook": "开头可以抓住人的冲突/反常识/困惑",
  "knowledge_gain": "观众听完获得的新框架或新理解",
  "resonance_points": ["至少2个被说中的现实处境"],
  "script_archetype": "从下方读书视频原型中选择一个",
  "main_thread": "贯穿全文的一句话主线",
  "evidence_boundary": "资料确定说了什么、没说什么、哪些不能夸大",
  "comment_tensions": ["3个真实讨论点，不是运营话术"]
}
```

## 读书视频原型

写之前先选一个主原型。原型不是模板，而是决定信息顺序和叙事重心。

### 1. 反常识解释型

适合资料推翻常见理解。结构重点是“大家以为 A，其实资料告诉我们 B，而 B 对现实有代价”。不要一开头就讲作者是谁。

### 2. 机制拆解型

适合经济、政治经济、制度、商业、社会科学资料。重点讲利益结构、激励机制、制度约束、代价与后果，解释钱、权、信用、政策、市场和普通人处境之间的关系。

### 3. 历史转折型

适合历史、人物、制度演变和时代变化。重点讲关键选择发生在什么约束下，为什么当时看起来合理，后来又如何改变结局。

### 4. 人物选择型

适合人物传记、组织史、案例研究。重点不是“这个人很厉害”，而是把他放回处境里：他看见了什么，能选什么，错过什么，承担什么。

### 5. 现实警示型

适合风险、危机、社会问题、AI 影响、公共议题。重点是把抽象风险落到工作、教育、治理、创新、安全和日常生活，避免恐吓式表达。

### 6. 方法启发型

适合认知科学、心理学、管理、方法论。每个核心点都要落到观众今天能理解或尝试的动作，同时说明适用边界、学习成本和常见失败点。

## 开头

- 前 100 字内不要寒暄、空话或背景铺垫。
- 第一句话必须提出一个牵引全文的问题、冲突或反常识判断；它必须来自原资料的真实矛盾或现实困惑。
- 前 100 字必须同时有问题/冲突和价值/收益承诺：让观众知道这段内容和自己有什么关系、听完能获得什么。
- 自然引出作品名 `《XX》`。
- 不要先讲作者生平，除非这对理解问题本身不可缺少。

可用开头方式：

- **荒诞事实**：先抛出资料里最反常识、最刺眼的事实。
- **现实困惑**：从观众也经历过但没说清的问题切入。
- **反向论证**：先承认常见看法，再说明这本书为什么把它翻过来。
- **历史回声**：用一个历史场景照到当下，但必须很快回到资料主线。

## 结构和节奏

- 围绕理解、留存和共鸣重组资料。
- 最值得听的内容放前面，不平均分配篇幅。
- 背景只为解释因果、结构、选择、约束、代价或后果服务。
- 约每 250 字要有一次认知推进：更尖锐的问题、反常识判断、案例/场景、现实对照、反转或代价。
- 高频写“扣主线句”。每次讲案例、概念或背景后，用一句话把它拉回 `_angle_brief.json` 的 `main_thread`。
- 知识要像“聊到这里自然想到”，不要写成“下面我来科普”。
- 避免连续两段只讲背景或概念。

推荐的口播推进方式：

- **层层剥开**：现象 -> 表面解释 -> 更深追问 -> 核心洞察。
- **逐一展示**：多个案例或观点不要一次性罗列，按弱到强、浅到深排列，每个都带一个判断。
- **人物画像**：从资料中的数据、制度或选择，快速代入一个具体普通人的处境，让抽象结论有体温。
- **文化升维**：结尾或关键转折处，可以连接到更大的历史、文化、哲学参照，但必须自然，不能硬拔高。
- **回环呼应**：开头埋下的问题、意象或判断，结尾要以变化后的视角回扣。

## 传播感

- 重要观点要落到具体的人、事、场景、制度、职业、生活选择、风险、机会或取舍。
- 至少写出两处“被说中”的感觉，让观众觉得自己经历过但没说清。
- 自然埋入三个评论触发点，来自现实分歧、经验对照、代价判断或价值张力。
- 至少有一句观点鲜明、好记、有记忆点的金句。
- 不制造廉价对立，不为了互动硬造问题。

评论触发点应该像真实判断：

- “真正难的不是知道答案，而是承认代价。”
- “有些选择不是对错问题，是你愿意把风险放在哪里。”
- “这本书最刺人的地方，是它把普通人的无力感讲清楚了。”

避免写成运营话术：

- “评论区告诉我”
- “你怎么看”
- “学到了扣1”
- “转发给有需要的人”

## 文风

- 使用简体中文。
- 口语化、大白话，适合直接口播。
- 每句尽量只有一个核心意思，避免抽象名词堆叠。
- 句子长短交替。需要强调时可以用短句；最终 JSON 的 `content` 保留自然段落换行，方便阅读和后续 DOCX 转录。
- 敢下判断，但不要装全知。表达要像“我读完之后更相信这个判断”，不是“我来给你上课”。
- 允许用“你可能也见过”“这就解释了为什么”等自然连接，但不要堆口头禅。
- 信息密度要高，但不要说教，也不要煽情过度。

## 内容类型适配

- 经济学 / 政治经济 / 制度分析：重点讲利益结构、激励机制、制度约束、代价与后果。
- 历史 / 历史解释 / 历史人物与结构：重点讲时代背景、关键选择、资源分配、制度演变和历史转折背后的真实因果。
- 认知科学 / 心理学 / 行为研究：重点讲结论、实验、机制、常见误区和适用边界，区分扎实研究、解释框架和仍有争议的部分。
- 社会理论 / 思想经典 / 社会科学名著：重点讲作者解释的社会现象、挑战的流行理解、提出的关键框架，以及它如何照到现实问题。
- 当代报告 / 机构研究 / AI 社会影响与公共议题：区分证据判断、推演、倡议和价值立场，优先落到就业、教育、治理、创新、安全和日常生活。

## 避免

不要使用这些表达或明显变体：

- “本书讲述了……”
- “作者认为……”
- “本书分为三部分……”
- “接下来我们来看……”
- “下面我们来介绍……”
- “总的来说……”
- “综上所述……”
- “值得注意的是……”
- “不难发现……”
- “引出了这本书里……”
- “这给我们的启示是……”
- “评论区告诉我”
- “你怎么看”
- “学到了扣1”
- “转发给有需要的人”

不要虚构“我亲身经历过”的第一人称经历。如果资料或用户没有提供真实经历，可以写观众处境、公共经验或资料中的人物和案例，但不要编造作者自己的故事。

## 结尾

结尾要重新钉住作品的核心价值，回扣开头的问题或意象，给观众一个现实中的思考方向、行动提醒或认知坐标，并自然留下一个适合继续讨论的判断或争议点。结尾要简洁、有余味，不要像课程收尾。

## 开场金句

- 金句不是标题，也不是总结句，要像开场前能让人停一下的判断。
- 用最短的句子，把一个复杂矛盾压缩成一个反差强烈的判断。能带来好奇和震撼。
- 不要写空泛鸡汤、营销口号、万能句。
- 每条 15-30 字，能单独成立，也能自然引出口播开头。

## 最终 JSON 输出契约

最终只输出严格 JSON，不要使用 Markdown，不要添加额外解释。若运行环境要求保存文件，保存到用户指定 raw JSON 路径，并确保 Python `json.loads` 可解析。

```json
{
  "source_name": "作品本身标题，需带书名号《》；不用文件名后缀",
  "video_titles": ["3条发布标题候选，互不重复，信息完整"],
  "cover_titles": ["3条封面主标题，4到10字，短、强、易扫读"],
  "cover_subtitles": ["3条封面副标题，15字以内"],
  "golden_quotes": ["3条开场金句，观点鲜明，像一句能让人停下来的真话"],
  "content": "完整口播终稿，按入口 SKILL.md 的 final_target_chars 控制长度，保留自然段落换行，不写Markdown或小标题",
  "total_length": 0
}
```

- `content` 由 `_draft_final.txt` 保留自然段落换行生成，可清理行首行尾空白，但不能改正文措辞。
- `total_length` 写实际 `content` 字符数。
- JSON 没有尾随逗号。
- 所有字符串正确转义；若直接输出 JSON 文本，`content` 中的段落换行必须写成合法的 `\n` 转义，不能在字符串里放未转义的物理换行。
- `content` 保留自然段落换行，但没有小标题或列表格式。
