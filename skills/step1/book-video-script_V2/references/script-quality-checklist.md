# Four-Layer Script Quality Checklist

这个清单检查对象是读书视频号/抖音口播稿。目标是把 AI 总结腔、读书报告腔和硬运营话术修掉。

在 `_draft_final.txt` 的一次终稿修订中完成四层质检。不要为了质检再新增一轮独立检查或单独报告文件。检查结果只需写入 `_revision_audit.json` 的 `quality_check_summary`。L1 和 L2 是硬闸门；L3 和 L4 发现关键问题时必须继续修 `_draft_final.txt`。

## L1 硬性规则检查

这一层检查绝对不能违反的规则。任何一项命中都必须修复。

### L1-1 禁用表达扫描

全文搜索以下表达或明显变体，出现则替换：

- “本书讲述了”
- “作者认为”
- “本书分为三部分”
- “接下来我们来看”
- “下面我们来介绍”
- “总的来说”
- “综上所述”
- “值得注意的是”
- “不难发现”
- “这给我们的启示是”
- “引出了这本书里”
- “评论区告诉我”
- “你怎么看”
- “学到了扣1”
- “转发给有需要的人”

### L1-2 格式扫描

- 最终 JSON 的 `content` 不得有 Markdown、小标题或列表；应保留自然段落换行。
- JSON 必须能被 `json.loads` 解析。
- `source_name` 必须是作品标题并带书名号《》。
- 每个候选数组必须各 3 条，互不重复。
- `total_length` 必须等于 `content` 实际字符数。

### L1-3 忠实性扫描

- 不虚构资料没有表达的立场。
- 不编造第一人称经历。
- 不把争议结论写成资料已经证明的事实。
- 不用资料中的边缘例子代表全书主张。
- 不为了金句牺牲准确性。

通过标准：L1-1 零命中，L1-2 全部合规，L1-3 没有硬伤。

## L2 口播结构检查

这一层检查稿子是否真的适合短视频口播。

### L2-1 开头检查

- 前 100 字内是否出现问题、冲突或反常识判断。
- 前 100 字内是否说明了观众为什么要听。
- 是否自然引出作品名《XX》。
- 是否避免作者生平、出版背景、目录介绍式开头。

### L2-2 主线和节奏检查

- `_angle_brief.json` 的 `main_thread` 是否贯穿全文。
- 每约 250 字是否有一次认知推进。
- 是否有高频回扣主线句，避免案例和概念散掉。
- 是否把最值得听的内容前置。
- 是否避免连续两段只讲背景或概念。

### L2-3 口播感检查

- 句子是否能顺口念出来。
- 是否拆掉了过长句、抽象名词堆叠和翻译腔。
- 是否像对一个普通朋友解释，而不是像课堂讲义。
- 是否有长短句变化，而不是整篇同一节奏。

### L2-4 读书报告腔检查

- 是否没有按章节平推。
- 是否没有用“作者认为”反复推进。
- 是否不是“观点一、观点二、观点三”的平面罗列。
- 是否从资料中重组出一个适合观众理解和留存的叙事。

通过标准：L2-1 全部通过，L2-2 至少 4 项通过，L2-3 至少 3 项通过，L2-4 全部通过。

## L3 内容质量检查

这一层检查内容是否有深度、有证据、有现实落点。

### L3-1 观点支撑

- 每个核心观点是否都有来自资料的人、事、场景、制度、实验、数据或原文论证支撑。
- 有没有空泛判断只有结论没有证据。
- 有没有把资料中的复杂判断过度简化成爽文金句。

### L3-2 知识输出方式

- 知识点是否像“讲到这里自然想起来”，而不是“下面开始科普”。
- 专业概念是否用普通人能理解的方式解释。
- 是否说明关键结论的适用边界。

### L3-3 现实共鸣

- 是否至少有两处“被说中”的现实表达。
- 抽象机制是否落到了职业、家庭、教育、消费、组织、制度、风险或选择。
- 是否有普通人的处境，而不只是宏大叙事。

### L3-4 原型专项检查

根据 `_angle_brief.json` 的 `script_archetype` 做专项检查：

- 反常识解释型：是否先承认常见理解，再揭示资料如何翻转它。
- 机制拆解型：是否讲清利益结构、激励、约束、代价和后果。
- 历史转折型：是否把人物和事件放回当时的资源、制度和信息约束。
- 人物选择型：是否讲清当事人能选什么、为什么那样选、付出什么代价。
- 现实警示型：是否避免恐吓，讲清风险如何落到普通生活。
- 方法启发型：是否有可执行动作、适用边界和失败点。

### L3-5 传播点检查

- 金句要简洁，又能带来好奇和震撼。
- 是否有三个自然评论触发点。
- 评论触发点是否来自真实价值张力，而不是运营口号。
- 标题、封面、金句是否互相区分，不是同一句话改写三遍。

通过标准：L3-1 和 L3-2 必须通过；L3-3 至少 2 项通过；L3-4 主原型相关项通过；L3-5 至少 3 项通过。

## L4 活人感终审

这一层以观众视角通读全文，回答一个核心问题：

**“听完这段，我感觉是一个读透资料、也理解普通人处境的人在跟我讲，还是一个 AI 在做总结？”**

### L4-1 姿态

- 语气是否平等，不像导师训话。
- 是否敢给判断，但没有装全知。
- 是否有人文关怀，而不是冷冰冰地输出信息。

### L4-2 心流

- 从头到尾是否顺着一个问题往前走。
- 有没有某一段突然偏离主线，需要观众重新找逻辑。
- 有没有背景太长、概念太密、节奏断掉的地方。

### L4-3 温度

- 是否能看见具体的人和选择，而不是只有抽象概念。
- 是否让观众觉得这件事和自己的生活有关系。
- 情绪是否来自资料矛盾和现实处境，而不是强行煽情。

### L4-4 作品感

- 开头埋下的问题或意象，结尾是否回扣。
- 全文是否像一个完整作品，而不是一串资料点。
- 金句、标题、评论点是否服务同一主线。

通过标准：整体感觉像真人讲稿。如果任何一项让你觉得“这段 AI 味太重”，就找出具体段落并返工。

## Audit 记录格式

把质检结果写入 `_revision_audit.json`：

```json
{
  "quality_check_summary": {
    "l1_hard_rules_passed": true,
    "l2_oral_structure_passed": true,
    "l3_content_quality_passed": true,
    "l4_human_feel_passed": true,
    "remaining_issues": []
  }
}
```

如果 L1 或 L2 为 `false`，不得包装最终 JSON，继续修 `_draft_final.txt`。如果 L3 或 L4 为 `false`，通常也应继续修稿；只有问题很小且不影响交付时，才允许在 `remaining_issues` 中说明残余风险。
